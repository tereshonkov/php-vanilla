# PHP Starter Project

## Как запустить

1. Соберите и запустите контейнеры:

   ```bash
   docker compose up -d

   Установите зависимости

   docker exec -it php-starter composer install

   Проверьте статус приложения:

   curl http://localhost:8080/health
   ```
### Принятые архитектурные решения

## Защита от RCE в Nginx: Конфигурация Nginx полностью блокирует прямой доступ к любым .php файлам, кроме index.php, возвращая ошибку 404. Это исключает запуск вредоносных скриптов, если они будут загружены на сервер.

## Инверсия зависимостей в PdoConnection: Класс PdoConnection избавлен от глобальных функций getenv(). Он получает конфигурацию строго через конструктор, что делает его тестируемым, независимым и предсказуемым.

## Строгая обработка ошибок: В index.php настроен глобальный обработчик set_exception_handler для \Throwable. Любые фатальные сбои или исключения будут перехвачены и отданы клиенту в формате JSON с корректным HTTP-кодом вместо падения сервера.

## Правильные HTTP-статусы для HealthCheck: Если база данных недоступна, роут /health возвращает HTTP-код 503 Service Unavailable, а не 200 OK. Это стандарт для систем мониторинга (например, Kubernetes).
